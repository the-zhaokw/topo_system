# TOPO系统 - 系统演示指南

## 演示概述

本演示将展示TOPO系统的核心功能，包括项目管理、Bug跟踪、知识库、用户管理、权限控制、个人计划与风险管理等。演示时长约45分钟。

## 演示准备

### 系统环境
- 确保后端服务运行在 http://localhost:5000
- 确保前端服务运行在 http://localhost:3000
- 准备测试数据（项目、Bug、用户、知识文章）

### 演示账户
- 管理员：admin / admin123
- 项目经理：project_manager / project123
- 开发人员：developer1 / dev123
- 测试人员：tester1 / test123

## 演示流程

### 第一部分：系统介绍（5分钟）

#### 1.1 登录系统
- 打开浏览器访问 http://localhost:3000
- 使用管理员账户登录
- 展示系统界面布局

#### 1.2 仪表板概览
- 展示项目统计信息
- 展示Bug状态分布
- 展示系统健康状态

### 第二部分：项目管理演示（10分钟）

#### 2.1 创建新项目
- 点击"项目管理" → "新建项目"
- 填写项目信息：
  - 项目名称："演示项目"
  - 描述："系统功能演示和培训用项目"
  - 状态：active
  - 成员：选择项目经理、开发人员、测试人员
- 演示项目创建成功

#### 2.2 项目编辑和成员管理
- 编辑项目信息
- 添加/移除项目成员
- 演示权限控制

#### 2.3 项目筛选和搜索
- 按状态筛选项目
- 按成员筛选项目
- 搜索功能演示

#### 2.4 项目风险与问题（v1.6 新增）
- 进入"风险与问题"页签
- 新建风险并设置概率/影响
- 展示敞口自动计算
- 关联到项目中的Bug

### 第三部分：Bug管理演示（10分钟）

#### 3.1 创建Bug报告
- 切换到测试人员账户
- 创建新的Bug报告：
  - 标题："购物车页面加载缓慢"
  - 描述：详细描述问题现象和重现步骤
  - 严重程度：高
  - 优先级：高
  - 分配人员：developer1
- 演示附件上传功能

#### 3.2 Bug状态流转
- 切换到开发人员账户
- 查看分配的Bug
- 将Bug状态改为"进行中"
- 完成修复后改为"已解决"

#### 3.3 Bug验证和关闭
- 切换回测试人员账户
- 验证Bug修复情况
- 关闭Bug或重新打开

#### 3.4 Bug筛选和搜索
- 按状态、严重程度、优先级筛选
- 高级搜索功能
- 批量操作演示

### 第四部分：知识库演示（5分钟，v1.6 增强）

#### 4.1 创建文章
- 创建多级分类
- 新建一篇 Markdown 文章
- 切换状态：草稿→已发布
- 收藏 / 置顶演示

#### 4.2 导出与分享
- 导出 PDF / Word / Markdown
- 生成外部分享 Token
- 通过链接访问（无登录）

#### 4.3 批量操作
- 勾选多篇文章
- 演示批量移动 / 状态 / 删除

### 第五部分：用户与权限演示（5分钟，v1.6 新增）

#### 5.1 模块权限
- 进入"模块权限"页面
- 选择一个普通用户
- 关闭部分模块演示菜单效果

#### 5.2 权限模板
- 创建"测试工程师"权限模板
- 批量应用到 3 名新员工
- 验证模块显示

### 第六部分：个人工作计划（5分钟，v1.6 新增）

#### 6.1 快速捕获
- 输入：`!紧急 明天下午3点 @开发 评审上线清单`
- 系统智能解析并落库

#### 6.2 四象限与专注
- 拖入四象限
- 启动番茄钟
- 完成 / 取消专注

#### 6.3 习惯追踪
- 创建每日习惯
- 演示连续打卡

### 第七部分：高级功能演示（5分钟）

#### 7.1 数据导出
- 导出项目列表（CSV/Excel）
- 导出Bug数据
- 演示导出文件内容

#### 7.2 统计分析
- 展示Bug趋势图表
- 项目进度分析
- 团队工作效率统计
- 合同、风险统计

## 演示技巧

### 成功演示的关键点
1. **准备充分**：提前测试所有功能流程
2. **数据真实**：使用有代表性的测试数据
3. **节奏控制**：保持适当的演示节奏
4. **互动参与**：鼓励观众提问和参与

### 常见问题准备
- 系统性能如何？
- 支持多少用户并发？
- 数据安全如何保障？
- 如何备份和恢复数据？
- 角色与模块权限有什么区别？
- 知识库如何控制外部分享安全？
