# 用户手册

本文档详细说明TOPO系统的使用方法，适合所有用户角色参考。

## 系统概述

TOPO系统是一个专业的组织架构管理与工作协同平台，帮助团队高效管理项目、跟踪Bug、管理物料、处理考勤、跟踪合同、管理需求和测试。

### 用户角色

| 角色 | 权限说明 |
|------|----------|
| **admin** | 系统最高权限，可以管理所有项目、用户和系统设置 |
| **user** | 普通用户，可以查看和操作系统功能 |
| **hr** | 人力资源管理员，负责考勤管理和审批 |
| **department_manager** | 部门经理，管理本部门人员和项目 |
| **division_leader** | 分管领导，高级管理人员 |
| **project_manager** | 项目经理，管理指定项目，可以创建项目、分配成员、查看统计 |
| **software_engineer** | 软件工程师，负责开发任务 |
| **test_engineer** | 测试工程师，负责测试任务 |
| **developer** | 开发人员，负责Bug修复，可以查看分配的任务、更新Bug状态 |
| **tester** | 测试人员，负责Bug报告和验证，可以创建Bug、验证修复结果 |

## 项目管理

### 创建新项目

1. 进入"项目管理"页面
2. 点击右上角"新建项目"按钮
3. 填写项目信息：

   - **项目代码**：必填，项目编码
   - **项目描述**：项目详细说明

1. 在项目列表中点击"删除"按钮

## Bug管理

### 创建Bug报告

1. 进入"Bug管理"页面
2. 点击"新建Bug"按钮
3. 填写Bug信息：

#### 基本信息
- **Bug标题**：简明扼要描述问题
- **Bug描述**：详细描述问题现象、重现步骤、期望结果
- **所属项目**：选择Bug关联的项目

#### 分类信息
- **严重程度**（Severity）：
  - trivial：微小问题
  - minor：次要问题
  - major：主要问题
  - low：低优先级
  - medium：中优先级
  - high：高优先级
  - critical：严重问题
  - blocker：阻止性问题
- **优先级**（Priority）：
  - low：可以延后处理
  - medium：正常处理优先级
  - high：需要尽快处理
  - critical：紧急处理

#### 分配信息
- **分配人员**：选择负责修复的开发人员
- **报告人员**：自动填充当前用户

### Bug状态管理

Bug状态流转图：
```
新建(new) → 已分配(assigned) → 进行中(in_progress) → 已修复(fixed)
                                                              ↓
关闭(closed) ← 已验证(verified) ← 已解决(resolved) ← ← ← ← ← ←
                    ↓
                重新打开(reopened)
```

#### 状态说明
- **新建(new)**：Bug刚被创建，等待分配
- **已分配(assigned)**：Bug已分配给开发人员
- **进行中(in_progress)**：开发人员正在修复
- **已修复(fixed)**：开发人员完成修复
- **已解决(resolved)**：等待测试验证
- **已验证(verified)**：测试人员验证通过
- **已关闭(closed)**：Bug正式关闭
- **重新打开(reopened)**：验证不通过，重新打开处理

### Bug操作指南

#### 开发人员操作
1. **接收Bug**：查看分配给自己的Bug
2. **开始处理**：将状态改为"进行中"
3. **完成修复**：将状态改为"已修复"，填写修复说明
4. **重新打开**：如果验证不通过，重新处理

#### 测试人员操作
1. **创建Bug**：发现问题时创建Bug报告
2. **验证修复**：检查已解决的Bug是否真正修复
3. **关闭Bug**：验证通过后关闭Bug
4. **重新打开**：验证不通过时重新打开

### Bug筛选和搜索

#### 快速筛选
- **按状态**：显示指定状态的Bug
- **按严重程度**：显示指定严重程度的Bug
- **按优先级**：显示指定优先级的Bug
- **按项目**：显示指定项目的Bug
- **按分配人**：显示分配给指定人员的Bug

#### 高级搜索
- **关键词搜索**：在Bug标题和描述中搜索
- **组合筛选**：多个条件组合筛选
- **时间范围**：按创建时间或更新时间筛选

### Bug统计

1. 进入"Bug统计"页面
2. 查看Bug分布图表
3. 分析Bug趋势

## 需求管理

### 需求文档管理

1. 进入"需求管理"页面
2. 创建需求文档：
   - **文档名称**：需求文档标题
   - **所属项目**：选择关联项目
   - **文档类型**：functional（功能）/non_functional（非功能）
   - **状态**：draft（草稿）/reviewing（评审中）/approved（已批准）/deprecated（已废弃）

### 需求条目管理

1. 在需求文档中创建需求条目：
   - **需求标识**：如REQ-001
   - **需求标题**：需求名称
   - **需求描述**：详细需求说明
   - **优先级**：1-低/2-中/3-高
   - **所属模块**：选择模块
   - **负责人**：指定负责人

2. 需求状态流转：
   - pending_review → reviewed → approved → in_progress → completed → verified

### 需求追溯矩阵

1. 进入"需求追溯矩阵"页面
2. 查看需求与Bug/任务/测试用例的关联关系
3. 支持追溯：
   - 需求 → 对应的测试用例
   - 需求 → 实现的Bug
   - 需求 → 关联的任务

### 需求仪表板

1. 进入"需求仪表板"查看需求概览
2. 查看需求状态分布
3. 查看需求与测试用例的关联率

## 测试管理

### 测试集管理

1. 进入"测试管理" → "测试集列表"
2. 创建测试集：
   - **测试集名称**：测试集标题
   - **所属项目**：选择项目
   - **类型**：functional（功能）/performance（性能）/security（安全）
   - **优先级**：1-低/2-中/3-高
   - **负责人**：指定负责人

### 测试用例管理

1. 在测试集中创建测试用例：
   - **用例标识**：如TC-001
   - **用例标题**：用例名称
   - **前提条件**：执行前置要求
   - **测试步骤**：详细操作步骤
   - **预期结果**：期望的系统行为
   - **优先级**：P0/P1/P2/P3

2. 测试用例状态：
   - designing（设计中）
   - 待评审
   - 已评审
   - 已批准
   - 废弃

### 测试执行

1. 进入"测试执行"页面
2. 创建测试执行记录：
   - **执行名称**：如"回归测试-2026-03-25"
   - **执行环境**：测试环境信息
   - **测试版本**：被测版本
   - **选择测试集**：选择要执行的测试集

3. 执行测试并记录结果：
   - passed（通过）
   - failed（失败）
   - blocked（阻塞）
   - skipped（跳过）

### 测试报表

1. 进入"测试报告"页面
2. 查看测试执行结果汇总
3. 导出测试报告

### 测试仪表板

1. 进入"测试仪表板"查看测试概览
2. 查看测试用例覆盖率
3. 查看测试执行趋势

### 风险管理

#### 创建风险/问题
1. 进入项目详情页的"风险与问题管理"
2. 点击"新建风险/问题"按钮
3. 填写风险信息：
   - **标题**：风险简明描述
   - **类型**：风险(Risk)或问题(Issue)
   - **等级**：low/medium/high/critical
   - **优先级**：low/medium/high/critical
   - **概率/影响**：0-1之间的小数
   - **敞口**：自动计算 (概率×影响)
   - **缓解策略**：风险应对措施
   - **应急预案**：备选方案

#### 风险状态管理
- identified：已识别
- analyzing：分析中
- mitigating：处理中
- resolved：已解决
- closed：已关闭

#### 风险矩阵
1. 进入风险列表页面
2. 查看风险矩阵视图
3. 按等级和优先级分类显示
4. 快速识别高风险项

#### 风险统计
1. 查看风险统计概览
2. 按状态/等级/类别分析
3. 导出风险报告

#### 关联管理
1. 将风险关联到Bug
2. 将风险关联到任务
3. 建立风险之间的关系

## 常见问题解答

### Q: 忘记密码怎么办？
A: 请联系系统管理员重置密码。

### Q: 如何查看我负责的Bug？
A: 在Bug管理页面使用"分配给我"筛选条件。

### Q: Bug状态流转有什么限制？
A: 只有开发人员可以将Bug改为"进行中"和"已修复"，只有测试人员可以关闭Bug。

### Q: 如何导出特定时间段的Bug数据？
A: 使用时间筛选功能后，再执行导出操作。

### Q: 附件上传失败怎么办？
A: 检查文件大小是否超过16MB限制，或联系管理员检查系统配置。

### Q: 如何创建需求与Bug的关联？
A: 在Bug详情页面，可以关联到对应的需求条目。

### Q: 测试用例如何关联需求？
A: 在测试用例详情页面，可以添加需求关联。
